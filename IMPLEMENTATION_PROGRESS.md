# DermaHealth — Implementation Progress

Tracks the gap-closing work described in the task brief: turning the frontend-only prototype documented in `FE_OVERVIEW.md` into a repository that demonstrates the full target workflow (Patient Intake → AI Preliminary Assessment → Doctor Review/Diagnosis → Clinical Orders → BPM Workflow → EMR → CRM Post-Visit Care → Escalation/Follow-Up Encounter) using typed mock repositories/services instead of a real backend.

## Phase 2 — Enterprise UI/UX redesign

A second pass replaced the hand-rolled visual layer (inline-styled divs, emoji-as-icons, hand-drawn SVG charts, native-HTML Kanban, ASCII workflow diagrams) with a production-grade medical design system, while leaving every service call, repository, and business rule from Phase 1 untouched — no page mutates a repository directly; all writes still go through `domain/services/*`.

**Libraries installed:** `antd` + `@ant-design/icons` (icons ultimately not used — see below), `highcharts` + `highcharts-react-official`, `@dnd-kit/core` + `@dnd-kit/sortable` + `@dnd-kit/utilities`, `@xyflow/react`.

**Design system:** `src/index.css` rewritten around the medical-blue token set (`--medical-blue-50…950`, `--surface-*`, `--text-*`, `--border-*`, flattened `--shadow-card`, reduced radii `--radius-xs…xl`); legacy token names (`--primary`, `--bg`, `--card`, etc.) point at the new values so nothing broke mid-migration. `src/theme/antdTheme.ts` maps the same palette onto AntD's `ConfigProvider` tokens (colors, radii, component-level overrides for Button/Card/Layout/Menu/Table/Tabs) so the app doesn't read as default AntD. `src/charts/highchartsSetup.ts` registers `highcharts-more`/`solid-gauge`/`accessibility`/`exporting` once and sets a shared flat/desaturated chart theme — pages import it instead of touching `Highcharts.setOptions` themselves.

**Icon system:** standardized on `lucide-react` everywhere (already the incumbent system from Phase 1); `@ant-design/icons` was installed per the brief but not imported anywhere to avoid mixing two icon systems within the same app. Zero emoji or Unicode-arrow/checkmark/bullet characters remain as icon substitutes anywhere in `src/pages` or `src/layouts` (verified by a full-repo scan); the handful of prose arrows found in FAQ copy and "before → after" labels were reworded or replaced with a real `ArrowRight` icon.

**Per-surface highlights:**
- Shell: `AppShell`/`Sidebar`/`TopHeader` rebuilt on AntD `Layout`/`Menu`/`Select`/`Popover`/`Badge` — role-aware navigation now renders as a real connected `Layout.Sider`, not a disconnected fixed div.
- Dashboard: AntD `Statistic`/`Card`/`Progress` grid + a Highcharts area chart replacing the hand-drawn SVG; patient and operational (per-role) variants both redone.
- Journey: AntD `Steps` (vertical milestone timeline), `Descriptions`, `Timeline` (event log), `Table` (orders) replacing the manual dot-and-line timeline markup.
- AI Analysis / Doctor Review: AntD `Steps` + `Upload.Dragger` + `Segmented`/`Checkbox.Group` for intake, `Result` for emergency/insufficient-data states, `Progress`-bar confidence bands.
- Work Queue: real drag-and-drop Kanban via `@dnd-kit/core` (`useDraggable`/`useDroppable`/`DragOverlay`, `PointerSensor` + `KeyboardSensor`) — every drop still calls the same `workflowService` transition functions as before, so an invalid drop throws `InvalidTransitionError` and surfaces as an AntD `message.error` in Vietnamese instead of silently succeeding.
- BPM workflow pages: template step graph and running-instance task graph both rendered with `@xyflow/react` (`ReactFlow`/`Background`/`Controls`/`MiniMap`, custom node components, auto-layout by prerequisite depth in the new `domain/flowLayout.ts`); template step reordering uses `@dnd-kit/sortable` (`SortableContext`/`arrayMove`) wired to `workflowService.reorderSteps`.
- Records: treatment-plan tab is now a `@dnd-kit` Kanban (3 columns, drag between them); EMR tab uses AntD `Descriptions`/`List`/`Alert` for the record, document, and audit views.
- Care, Audit Viewer, Integrations, Appointments, Profile, Prescriptions, Progress, Reports, Settings, Support, Login: all converted from inline-styled `<div>`/`<button>`/`<select>` markup to the corresponding AntD components (`Table`, `Collapse`, `Calendar`, `Timeline`, `Tabs`, `Form`, `Switch`, `Menu`, etc.), with Progress/Reports' hand-drawn SVG charts replaced by Highcharts area/column charts.

**Validation:** `npm run build` and `npm run lint` clean after every file group (see inline history); all 18 active routes return HTTP 200 from the dev server. No browser-level visual verification was performed — this sandbox has no headless-browser tool and installing one wasn't requested, so correctness here rests on TypeScript, ESLint (including the `react-hooks` purity/memoization rules, which caught two real issues during this pass), and a full production bundle build, not a rendered screenshot.

**Known trade-off (resolved in Phase 3):** the production bundle grew from ~550KB to ~2.3MB (gzip ~740KB) with antd + Highcharts + React Flow + dnd-kit added, with no route-level code-splitting. Phase 3 below fixes this via `React.lazy`.

## Remaining frontend-prototype limitations

- `Appointments.tsx`, `Prescriptions.tsx`, `Progress.tsx`, `Reports.tsx` were not migrated onto the shared domain/repository layer — they still render their own local fixture arrays (per Task brief §5's list, these were lower-value targets than the clinical/operational pages and were deprioritized under time constraints). Values are textually consistent with the canonical patient/doctor identity but are not live-reactive to it.
- No dedicated `DigitalSignature` UI — signature capture is represented via `MedicalRecord.signedBy/signedAt` rather than a standalone signature-object flow.
- Browser-level UI verification was not performed in Phase 1/2 (no headless browser tool available in this sandbox and installing one wasn't requested); verification was build+lint clean plus manual code trace of every guard. Phase 3 below documents verification methods explicitly per claim — see `FINAL_VERIFICATION.md`.
- Several routes (`Appointments`, `Profile`, `Prescriptions`, `Progress`, `Reports`, `SettingsPage`'s notification/privacy/device toggles, `Support`, most of `Dashboard`'s patient view, parts of `Care`) render static local-fixture data or no-op buttons rather than live repository data — see `FINAL_VERIFICATION.md` for the exact per-route breakdown. This was already the case going into Phase 3 and was out of scope for a hardening pass (no business-logic/feature work), so it is called out here rather than silently left undocumented.
- Only 3 of ~19 in-app routes (`DoctorReview`, `AuditViewer`, `Integrations`) enforce an actual page-level role block (`if (role !== ...) return <Result/>`). All other role-restricted pages rely solely on the Sidebar hiding the nav link — navigating directly by URL bypasses this. There is no backend, so this is a client-side-only convenience gate, not real access control; see `FINAL_VERIFICATION.md` §RBAC.

## Phase 3 — Production hardening pass

Scope: harden the Phase 2 visual baseline for correctness/safety without redesigning it again or touching domain services/repositories/business rules, per an explicit 14-point brief. All 12 work items below are complete; `npx tsc -b --noEmit` and `npx eslint .` are clean after every item (re-verified together at the end of the pass).

1. **`@ant-design/icons` audit** — confirmed zero imports anywhere in `src` (lucide-react is the sole icon system, as already stated above); ran `npm uninstall @ant-design/icons` to drop it from `package.json` (still available transitively via `antd`'s own dependency).
2. **Route-level code splitting** — every routed page in `src/App.tsx` converted to `React.lazy(() => import(...))`; `AppShell`, `Sidebar`, `TopHeader`, `AppStateProvider`, and the guard/error-boundary layer stay eager. Two `Suspense` boundaries: one at `App.tsx` top level (`FullPageLoadingFallback`, a centered `Spin`) for pre-shell routes, one inside `AppShell.tsx` around `<Outlet/>` (`ContentLoadingFallback`, an AntD `Skeleton`) so the sidebar/header stay mounted during in-app navigation. New file: `src/layouts/RouteFallback.tsx`.
3. **Global `ErrorBoundary`** — new `src/layouts/AppErrorBoundary.tsx` (class component, `componentDidCatch`/`getDerivedStateFromError`), wraps the whole app inside `ConfigProvider`/`AntApp` in `App.tsx`. Module init (`highcharts-more`, `highcharts-3d`, `solid-gauge`, `accessibility`, `exporting`) happens exactly once, client-side only (Vite SPA, no SSR path exists), and is not duplicated per page.
5. **React Flow hardening** — `WorkflowTemplateEditor.tsx` and `WorkflowInstancePage.tsx` invalid/missing-ID states upgraded from a plain-text `<Card>` to an AntD `Result` with the `SearchX` icon, the actual invalid route id echoed back in Vietnamese, and a button back to a safe list route (`/app/workflows/templates` / `/app/work-queue`) via `useNavigate` — confirmed these early returns sit after every hook call (no Rules-of-Hooks violation).
